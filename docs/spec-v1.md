# 決まり字トライ — 実装仕様書 v1.0

**読者**：実装を担当するAI（Claude Opus等）および将来の自分。
**この文書の規範性**：§2〜§4（数学的コア・データ形式・アーキテクチャ）は**規範**であり、変更には設計判断の記録が必要。§5（ビュー仕様）は**暫定**——UI/UXは今後改善される前提で、現時点の決定値を記す。§7の決定記録を再審理しないこと。

---

## 0. プロジェクトの一行定義

有限の語彙を、競技かるたの「決まり字」の構造で探索する道具。
語彙 W を文字（原子）のトライとして空間化し、「そこで一語に定まる点」を主役にする。

**3つのコア**（これに反する機能は追加しない）
1. シンタクティックに語を見る — 意味なしで全機能が成立する
2. 数学的に依存構造を見る — 実装は等式で検証できる純関数の集まり
3. 論理が空間的に見える — 多義・未知・不整合は「エラー」ではなく「見える状態」

**非目標**：学習進捗・SRS・通知・自動形態素解析・サーバー。「眺める道具」を守る。

---

## 1. 全体像（3分で掴む）

- ユーザーが触るのは **JSONファイル2種類**：語彙（単語リスト）と辞書（形態素の意味表）。
- アプリは5層：L0語彙 → L1トークナイザ(ソケット) → L2カーネル(不変) → L3辞書(ソケット) → L4ビュー。
- **トライビューが主**（L0〜L2だけで動く）。セマンティックビューはおまけ（L3が要る）。
- 真実は語彙Wただひとつ。トライ・count・決まり字・順序・レイアウトは全部Wの純関数。
- 参照実装：`kimariji-m1.jsx`（アーティファクト版プロトタイプ）。カーネル部は概ね移植可能。

## 2. 数学的コア【規範】

### 2.1 定義

- **原子集合 Σ**：トークナイザが定める。カーネルはΣの中身を一切知らない。
- **語彙 W ⊆ Σ***：有限集合。原子列の集合。
- **接頭辞順序**：p ⊑ q ⟺ pはqの接頭辞。P(W) ＝ Wの接頭辞閉包は有限半順序集合であり、
  任意の2元がmeet（最長共通接頭辞）を持つ**meet半束**。最小元はε。
- **トライ**：P(W)のHasse図。実装上は A^(Σ*) ≅ A × (A^Σ*)^Σ の同型が与える
  再帰構造 `Node = { value, children: Σ ⇀ Node }`（Hinzeのmemo trie）。
- **count : P(W) → ℕ**：`count(p) = |{w ∈ W : p ⊑ w}|`。接頭辞順序に関して**反調和**
  （p ⊑ q ⇒ count(p) ≥ count(q)）。
- **決まり字**：`kimariji(w) = min{ p ⊑ w : count(p) = 1 }`。
  鎖上でcount=1は上に閉じるため、この最小元（閾値）は常に一意に存在する。
- **子の全順序**：count降順、同数は collator（manifest指定）昇順。決定的。
- **焦点の修復**：語彙編集後、旧焦点fの新しい焦点は「fの接頭辞のうちP(W')に残る最大元」
  ＝ meetによるフォールバック。

### 2.2 状態

アプリの可変状態は最小に保つ：

```
State = {
  vocab: Entry[]          // 語彙（真実）
  dicts: Dict[]           // 有効な辞書、順序つき
  focus: Prefix           // 現在の接頭辞（トライ）／形態素ID（セマンティック）
  rot:   number           // 文字盤の回転角
  selection: WordId | null
  tab: "trie" | "semantic" | "archive"
}
```

推奨（必須ではない）：vocabへの変更をイベント列（追加/削除/seg更新/ピン留め）で保持し、
`state = fold(update)(init, events)` として導出する（イベントソーシング）。
undo＝末尾を除いて畳み直し。「語彙の成長と決まり字地形の変化」の再生が可能になる。

### 2.3 法則（プロパティテストとして実装すること）

実装の受け入れ条件。ランダム生成した語彙・操作列で検証する：

1. **トークン結合則**：`join(tokenize(w)) === w`（切断の可逆性）
2. **count反調和**：p ⊑ q ⇒ count(p) ≥ count(q)
3. **決まり字＝閾値**：kimariji(w)より真に短い接頭辞のcountは常に≥2、以降は常に1
4. **順序の決定性**：同じWから常に同じ描画順（collatorタイブレークの検証）
5. **編集の往復**：add(w)してremove(w)した語彙は元と外延的に等しい
6. **焦点修復**：修復後の焦点は必ずP(W')の元であり、旧焦点の接頭辞の中で最大
7. **レイアウトの純粋性**：同じ(W, focus, rot, 寸法)から同じ座標（乱数・時刻を使わない）

バネの実行時状態（位置・速度）は導出値のアニメーションであり、Stateに含めない。

## 3. データ形式【規範】

### 3.1 語彙ファイル

```json
{
  "manifest": {
    "format": 2,
    "name": "英動詞B2",
    "tokenizer": "grapheme",
    "collation": "en",
    "dictionaries": ["std-en-latin@1"]
  },
  "entries": [
    { "id": "11010", "w": "extract",     "seg": ["ex", "tract"],         "g": "抽出する" },
    { "id": "10100", "w": "composition", "seg": ["com", "pose", "tion"], "g": "構成" },
    { "id": "10240", "w": "stand",                                        "g": "立つ" }
  ]
}
```

規則：
- **必須は `w` のみ**。他は全て任意。無いフィールドの機能は静かに眠る。
- `id`：安定参照用。省略時は読み込み時に自動採番（採番結果は保存する）。
- `seg`：**表層分割**（文字列の分解＝構文側の事実）。要素は文字列、または
  ピン留め形 `{ "s": "co", "m": "math-en/co-compl" }`。混在可。
- `tokenizer`："grapheme"（Intl.Segmenter）／"codepoint"／
  `{ "type": "table", "digraphs": ["ch","ng"] }`。エントリ側の `atoms` 配列明示も許す
  （存在すればトークナイザより優先）。
- **未知のフィールドは保存時に削除しない**（前方互換）。
- `format` による移行：旧形式（M1の素配列）は読み込み時に v2 へ自動変換。
- 将来の機能追加は本体スキーマを変えず、**idをキーにしたサイドカーJSON**で行う。

### 3.2 辞書ファイル（プラグイン）

```json
{
  "meta": { "id": "std-en-latin", "version": 1, "name": "英ラテン標準", "author": "idroty" },
  "morphs": [
    { "id": "con",   "kind": "prefix", "forms": ["con","com","cor","col","co"], "gloss": "共に" },
    { "id": "tract", "kind": "root",   "forms": ["tract"],                       "gloss": "引く" },
    { "id": "tion",  "kind": "suffix", "forms": ["tion","sion"],                 "gloss": "名詞化" }
  ]
}
```

- `kind` ∈ {prefix, root, suffix}（セマンティックビューの層の割り当てに使う）。任意。
- 辞書は複数有効化でき、**順序つき**（上が優先＝既定の読み）。

### 3.3 seg解決の4状態（エラーは存在しない）

| 状態 | 条件 | 挙動 |
|---|---|---|
| 一意 | 表層形が有効辞書のformsに1件だけ一致 | その読みを表示 |
| 多義 | 複数辞書/形態素に一致 | 詳細画面に全候補を並記。語ごとにピン留め可 |
| 未知 | どの辞書にも無い | 灰色表示。タップでその場で辞書に登録できる |
| 不整合 | segの連結 ≠ w | バッジ表示し、セマンティック層から黙って除外 |

不整合でも**トライには常に載る**（segが構文を壊すことは原理的にない）。

## 4. アーキテクチャ【規範】

### 4.1 ファイル構成

```
kimariji-trie/
├─ data/
│  ├─ vocab/   … 語彙JSON（複数。例：en-b2-verbs.json, hyakunin.json）
│  └─ dicts/   … 辞書JSON（std-en-latin.json, math-en.json, …）
├─ src/
│  ├─ core/    … 純粋。言語もReactもDOMも知らない
│  │   ├─ trie.ts      … buildTrie, count, only, nodeAt, sortedKids
│  │   ├─ kimariji.ts  … kimarijiDepth, 深度分布
│  │   ├─ order.ts     … collatorタイブレーク
│  │   ├─ layout.ts    … 縦列/文字盤/魚眼トレイルの座標（純関数）
│  │   └─ resolve.ts   … seg×辞書の解決（4状態）
│  ├─ adapters/… 世界に触る側
│  │   ├─ tokenizer.ts … grapheme / codepoint / table / atoms
│  │   ├─ storage.ts   … localStorage ⇄ window.storage の切替
│  │   └─ files.ts     … JSON読み書き・formatの自動移行・未知フィールド保全
│  ├─ views/   … Stateを絵にする純関数。書き込まずMsgを投げるだけ
│  │   ├─ TrieView.tsx / SemanticView.tsx / DetailSheet.tsx
│  │   ├─ ArchiveView.tsx（語彙・辞書・インポート・設定）
│  │   └─ springs.ts   … 減衰振動子（表示専用）
│  ├─ state.ts … State型・update(msg, state)・イベント列
│  └─ App.tsx  … 配線のみ
└─ docs/       … 本仕様・スケッチ・決定記録
```

### 4.2 依存規則

- 依存方向は `views → core ← adapters` の一方向（非循環）。
- core はいかなる import も UI/IO 側へ向けない。coreの単体テストはDOM不要で走ること。
- views は state を読むだけ（CQRS）。書き込みは update 経由のMsgのみ。
- **ビューを足す＝views/に1ファイル。言語を足す＝data/に1ファイル。**
  **機能を消す＝ファイルを消す。**（SemanticView削除でトライは無傷、を保つ）

### 4.3 配布

Vite + React + TypeScript。GitHub Actions → GitHub Pages。PWA（manifest＋service worker、
オフライン動作、ホーム追加）。状態のURLハッシュ共有（`#/cond`）。
storageアダプタ：本番=localStorage、アーティファクト=window.storage。

## 5. ビュー仕様【暫定：UI/UXは今後改善】

### 5.1 共通

- タブ3つ（上部固定のセグメント）：**トライ／セマンティック／書庫**。
  選択ピルはバネで滑る。書庫はアイコン主体で視覚的重みを一段落とす。
- 配色トークン：紙#FCFBF9／線#C9BEB8／ローズ#C99B95（決まり字・選択形態素）／
  濃ローズ#B3554A（焦点の輪）／墨#6D625C／青#4DA3E8（選択語・n字決まり表示のみ）。
- 色の文法：**塗り＝決まった札、輪＝いま居る場所**。混ぜない。
- prefers-reduced-motion でバネを即時整定に切替。

### 5.2 トライビュー

- **語頭（focus=ε）は縦一列**。頻度順に上から。スクロール可。文字盤にしない
  （εは「何も共有していない語たちへの分配」であり、盤とは関係の種類が違う）。
- **深さ≥1は文字盤**：焦点の子を半径R_FAN=118の弧上に、頻度1位を-50°（1時の位置）から
  25°刻みで+方向へ。スワイプ＝盤の連続回転（rot、クランプ付き）。
  可視域|θ|≤63°、縁でフェード、**縁の外の次候補1〜2個を文字なし点線円で覗かせる**（P2）。
  薄い弧ガイド線を描く。
- **子ノード半径は控えめに**（現行baseRadiusより2〜3px小さく）。
  当たり判定は透明円で44px相当を維持（見た目と分離）。
- **エッジ幅＝count**：`w = 1 + 2.2·log(c)/log(c_max)`、上限3.2px、色は変えない（P3）。
- **トレイル＝魚眼**：等比S_TRAIL=0.6（総幅がSTEP×1.5で有界＝深さ10でも必ず画面内）。
  折れの向きは「選んだ子の順位オフセット」を保存（実木のズームである、という原則）。
  **文字サイズに下限10px**、円からはみ出してよい。奥は円が点になっても文字列は読める（P1）。
- 決まり字の子＝ローズ塗り。タップで選択。**尾は小さなゴースト文字連鎖のみ**、
  終端に単語テキストは置かない（語は下部シートで）（P1）。
- 姉妹ゴースト：トレイル各ノードに、選ばなかった枝を実座標の縮小で±3順位まで。
  タップで枝替え。トレイルノードのタップ＝巻き戻り。
- バネ定数（現行値、調整可）：K=110, D=14, WOBBLE=240。タップしたノードは左右にひと揺れ。
  新規ノードは親の現在位置から生まれる。

### 5.3 セマンティックビュー（二層〜多層グラフ）

- **ノード＝形態素**（辞書の住人）。列＝kind：接頭辞｜語根｜接尾辞。
- **エッジ＝語**（語彙の住人）。segに沿って層を渡る**道**。
  2形態素なら1本、3形態素なら折れ線。接尾辞の無い語の道は語根で終わる。
- **選択は状態であって出来事ではない**。時限グロー禁止。選択中の語の道と両端だけ
  太く・塗りでくっきり、他は薄く沈める。解除＝別の選択 or 背景タップ。
- トライからのジャンプ＝「その語の道が選択済みの初期状態」でこの画面を開くだけ。
  開いた瞬間、道が画面内に来るよう縦位置を合わせる。
- 形態素タップ＝そこを通る道を全て強調（ファイバー）。語の道タップ＝選択。
  シートから「◔トライへ」で往復。

### 5.4 書庫

語彙一覧（使用中マーク・編集・書き出し・インポート）、辞書一覧（優先順の並び替え・
有効/無効・インポート）、引き当て集計（一意/多義/未知/不整合→一覧へ）、
表示設定（揺れ強度、reduced-motion尊重）。主役2画面に歯車を置かない。

## 7. 決定記録（再審理しない）

1. 子の順序＝count降順・collator昇順タイブレーク（アルファベット固定順ではない）
2. 扇の角度は均等。頻度は「半径」と「線の太さ」で語る（角度配分はしない）
3. トレイル＝実木の魚眼ズーム。折れの向き＝兄弟内順位（人工的な対角線は不採用）
4. 語頭だけ縦一列（形の一貫性より意味の一貫性）
5. 焦点＝白地＋濃ローズの輪、決まり字＝ローズ塗り（色の文法を混ぜない）
6. スワイプ＝盤の回転（候補の入れ替えではない）
7. seg＝語彙側の構文の事実、辞書＝交換可能な解釈（構文/モデルの分離）。
   cotangentの多義は並記＋ピン留めで扱う。正解は主張しない
8. エラー概念の不採用：欠落・不整合・未知・多義は全て表示される状態
9. セマンティック＝形態素の層＋語の道。選択は永続のくっきり強調、時限演出なし
10. 単語の終端テキストはトライ上に置かない（下部シートに一本化）
11. 保存は本番localStorage／試作window.storage。書き出したJSONがdata/の正本
12. 求心式ナビゲーション採用（パン操作の原則廃止）。カメラ移動＝バネ1本
